Les groupes

Les groupes ont deux fonctions :


Rassembler un ensemble de contraintes ensembles pour les quantifier/appliquer une logique

Capturer des valeurs
Rassembler les contraintes :

/ab+/ et /(ab)+/ sont interprétés différemment

Le premier est un a suivi du plus possible de b, avec au moins 1 b

Le second est un a suivi d'un b, le tout répété le plus possible de fois
On peut aussi faire un "OU" avec des groupes

/ab|cd/ et /a(b|c)d/ sont interprétés différemment

Le premier donne ab ou cd

Le second donne a et (b ou c) et d
La capture en regex

Par défaut, les groupes sont "Capturants"

Une regex permet de matcher un texte avec des contraintes données

Les groupes capturants permettent d'affiner des petits sous-ensemble et d'extraire de l'information
La capture en regex

Exemple : /\w+?@(\w+\.\w+)/ va matcher un email simple et capturer le nom de domaine
La capture en regex

Les captures peuvent ensuite être utilisées :

Dans votre code
On peut aussi directement donner un texte de remplacement, comme sur Notepad++
Ensuite dans le texte de remplacement, les captures sont numérotées
Pour les remplacer dans un texte : $#
# étant un nombre

$0 représente le match complet
$1 représente la 1ère capture
$2 la 2ème, ...etc

Exemple de remplacements
Parfois aussi on souhaite avoir un groupe sans matcher
La syntaxe est alors : (?:)
Le groupe devient non-capturant
